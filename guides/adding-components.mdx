TailAdmin's UI is made up of standard React components styled with Tailwind CSS v4 utility classes. When you create a new component, follow the same conventions to keep the design consistent.

## Component anatomy

A typical TailAdmin component:

1. Is a React functional component written in TypeScript.
2. Accepts typed props via an `interface`.
3. Uses Tailwind utility classes directly in JSX — no CSS modules or `styled-components`.
4. Applies dark-mode styles using the `dark:` prefix.
5. Uses responsive prefixes (`sm:`, `md:`, `lg:`, `xl:`) for layout breakpoints.

## Creating a stat card widget

The following example follows the same pattern used by `EcommerceMetrics` in `src/components/ecommerce/EcommerceMetrics.tsx`.

```tsx
// src/components/common/StatCard.tsx
import React from "react";
import { ArrowUpIcon, ArrowDownIcon } from "@/icons";

interface StatCardProps {
  label: string;
  value: string;
  trend: "up" | "down";
  trendValue: string;
  icon: React.ReactNode;
}

export const StatCard: React.FC<StatCardProps> = ({
  label,
  value,
  trend,
  trendValue,
  icon,
}) => {
  return (
    <div className="rounded-2xl border border-gray-200 bg-white p-5 dark:border-gray-800 dark:bg-white/[0.03] md:p-6">
      <div className="flex items-center justify-center w-12 h-12 bg-gray-100 rounded-xl dark:bg-gray-800">
        {icon}
      </div>
      <div className="flex items-end justify-between mt-5">
        <div>
          <span className="text-sm text-gray-500 dark:text-gray-400">
            {label}
          </span>
          <h4 className="mt-2 font-bold text-gray-800 text-title-sm dark:text-white/90">
            {value}
          </h4>
        </div>
        <span
          className={`inline-flex items-center gap-1 rounded-full px-2.5 py-0.5 text-theme-xs font-medium ${
            trend === "up"
              ? "bg-success-50 text-success-600 dark:bg-success-500/10 dark:text-success-400"
              : "bg-error-50 text-error-600 dark:bg-error-500/10 dark:text-error-400"
          }`}
        >
          {trend === "up" ? <ArrowUpIcon /> : <ArrowDownIcon />}
          {trendValue}
        </span>
      </div>
    </div>
  );
};
```

Use it on a page:

```tsx
import { StatCard } from "@/components/common/StatCard";
import { GroupIcon } from "@/icons";

<div className="grid grid-cols-1 gap-4 sm:grid-cols-2 md:gap-6">
  <StatCard
    label="Customers"
    value="3,782"
    trend="up"
    trendValue="11.01%"
    icon={<GroupIcon className="text-gray-800 size-6 dark:text-white/90" />}
  />
</div>
```

## Tailwind CSS v4 utility classes

Tailwind CSS v4 is configured entirely through `src/app/globals.css` — there is no config file to edit. The custom design tokens defined in the `@theme` block are available as utility classes everywhere:

| Token type | Example token | Utility class |
|---|---|---|
| Brand color | `--color-brand-500` | `bg-brand-500`, `text-brand-500` |
| Gray color | `--color-gray-200` | `border-gray-200`, `bg-gray-100` |
| Success color | `--color-success-500` | `text-success-600` |
| Text size | `--text-theme-sm` | `text-theme-sm` |
| Shadow | `--shadow-theme-md` | `shadow-theme-md` |

## Dark mode

Dark mode is toggled by adding a `dark` class to `<html>` (managed by `ThemeContext`). Apply dark styles alongside light styles using the `dark:` prefix:

```tsx
// Light: white background, dark border
// Dark: semi-transparent white background, darker border
<div className="bg-white border border-gray-200 dark:bg-white/[0.03] dark:border-gray-800">
```

Common dark-mode patterns used across TailAdmin components:

| Element | Light class | Dark class |
|---|---|---|
| Card background | `bg-white` | `dark:bg-white/[0.03]` |
| Card border | `border-gray-200` | `dark:border-gray-800` |
| Primary text | `text-gray-800` | `dark:text-white/90` |
| Secondary text | `text-gray-500` | `dark:text-gray-400` |
| Icon container | `bg-gray-100` | `dark:bg-gray-800` |

## Responsive design

TailAdmin uses a mobile-first approach. The default classes apply on all screen sizes; prefixed variants override at larger breakpoints:

```tsx
// padding: 20px on mobile, 24px on md+ screens
<div className="p-5 md:p-6">

// single column on mobile, two columns on sm+
<div className="grid grid-cols-1 gap-4 sm:grid-cols-2 md:gap-6">
```

The custom breakpoints defined in `globals.css`:

| Prefix | Breakpoint |
|---|---|
| `2xsm:` | 375px |
| `xsm:` | 425px |
| `sm:` | 640px |
| `md:` | 768px |
| `lg:` | 1024px |
| `xl:` | 1280px |
| `2xl:` | 1536px |
| `3xl:` | 2000px |

## Using icons

All icons are SVG files in `src/icons/` imported as React components via `@svgr/webpack`. They are all re-exported from `src/icons/index.tsx`:

```tsx
import {
  GridIcon,
  UserCircleIcon,
  PieChartIcon,
  ArrowUpIcon,
  ArrowDownIcon,
  BellIcon,
} from "@/icons";
```

Icons render as inline SVGs and inherit the current text color, so you style them with text-color utilities:

```tsx
<GridIcon className="text-gray-500 group-hover:text-gray-700 dark:text-gray-400" />
```

### Adding a custom icon

<Steps>
  <Step title="Add your SVG file">
    Drop your `.svg` file into `src/icons/`. For example: `src/icons/rocket.svg`.
  </Step>

  <Step title="Export it from the index">
    Open `src/icons/index.tsx` and add the import and export:

    ```tsx
    import RocketIcon from "./rocket.svg";

    export {
      // ...existing exports
      RocketIcon,
    };
    ```
  </Step>

  <Step title="Use it in a component">
    ```tsx
    import { RocketIcon } from "@/icons";

    <RocketIcon className="text-brand-500" />
    ```
  </Step>
</Steps>

## Custom utility classes

If a set of classes is repeated across multiple components, extract them into a `@utility` in `globals.css`:

```css
@utility stat-card {
  @apply rounded-2xl border border-gray-200 bg-white p-5
         dark:border-gray-800 dark:bg-white/[0.03] md:p-6;
}
```

Then use `stat-card` as a single Tailwind class anywhere in your JSX.
