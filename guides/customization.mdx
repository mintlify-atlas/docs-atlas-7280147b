title: "Customization"

TailAdmin is designed to be easy to adapt to your brand and product requirements. All design tokens live in a single CSS file, navigation is declared in a plain TypeScript array, and every UI component is a standard React component you can edit directly.

## 1. Colors

All color values are defined as CSS custom properties inside the `@theme` block in `src/app/globals.css`. Because TailAdmin uses Tailwind CSS v4, there is no `tailwind.config.js` — the CSS file is the single source of truth.

To change the primary brand color, update the `--color-brand-*` values:

```css
@theme {
  --color-brand-25:  #f2f7ff;
  --color-brand-50:  #ecf3ff;
  --color-brand-100: #dde9ff;
  --color-brand-200: #c2d6ff;
  --color-brand-300: #9cb9ff;
  --color-brand-400: #7592ff;
  --color-brand-500: #465fff;  /* primary interactive color */
  --color-brand-600: #3641f5;
  --color-brand-700: #2a31d8;
  --color-brand-800: #252dae;
  --color-brand-900: #262e89;
  --color-brand-950: #161950;
}
```

Replace the hex values with your own palette. The change propagates instantly to every component that uses a `bg-brand-*`, `text-brand-*`, or `border-brand-*` utility — no component edits required.

### Semantic palettes

The same pattern applies to the other palettes declared in `globals.css`:

| Palette | Typical usage |
|---|---|
| `gray-*` | Backgrounds, borders, neutral text |
| `success-*` | Positive states, confirmations |
| `error-*` | Destructive actions, validation errors |
| `warning-*` | Caution states |
| `orange-*` | Accent color |
| `blue-light-*` | Secondary informational color |

<Tip>
  See the [Theming](/configuration/theming) page for the complete list of color tokens and shadow tokens available out of the box.
</Tip>

## 2. Typography

TailAdmin loads **Outfit** from Google Fonts via `next/font/google` in the root layout.

```tsx
// src/app/layout.tsx
import { Outfit } from 'next/font/google';

const outfit = Outfit({
  subsets: ["latin"],
});

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body className={`${outfit.className} dark:bg-gray-900`}>
        {children}
      </body>
    </html>
  );
}
```

The font is also registered as a `@theme` token so that `font-outfit` is available as a Tailwind utility:

```css
@theme {
  --font-outfit: Outfit, sans-serif;
}
```

### Switching to a different font

<Steps>
  <Step title="Replace the import in layout.tsx">
    ```tsx
    import { Inter } from 'next/font/google';

    const inter = Inter({ subsets: ["latin"] });
    ```
  </Step>

  <Step title="Apply the new font class">
    ```tsx
    <body className={`${inter.className} dark:bg-gray-900`}>
    ```
  </Step>

  <Step title="Update the @theme token in globals.css">
    ```css
    @theme {
      --font-outfit: Inter, sans-serif;
    }
    ```

    You can rename `--font-outfit` to `--font-inter` (and update `font-outfit` usages throughout the project) or leave the token name as-is — it makes no difference to how the font renders.
  </Step>
</Steps>

### Custom text size tokens

TailAdmin adds display and UI text sizes on top of Tailwind's defaults:

```css
@theme {
  --text-title-2xl: 72px;
  --text-title-xl:  60px;
  --text-title-lg:  48px;
  --text-title-md:  36px;
  --text-title-sm:  30px;
  --text-theme-xl:  20px;
  --text-theme-sm:  14px;
  --text-theme-xs:  12px;
}
```

Use them as Tailwind utilities: `text-title-md`, `text-theme-sm`, and so on.

## 3. Sidebar navigation items

Navigation is driven by two static arrays in `src/layout/AppSidebar.tsx`: `navItems` (rendered under the **Menu** heading) and `othersItems` (rendered under the **Others** heading).

```tsx
type NavItem = {
  name: string;
  icon: React.ReactNode;
  path?: string;       // direct link — omit when using subItems
  subItems?: {
    name: string;
    path: string;
    pro?: boolean;     // shows a "pro" badge
    new?: boolean;     // shows a "new" badge
  }[];
};
```

To add a direct-link item:

```tsx
import { GridIcon } from "../icons/index";

const navItems: NavItem[] = [
  // ...existing items
  {
    icon: <GridIcon />,
    name: "Analytics",
    path: "/analytics",
  },
];
```

To add a collapsible group:

```tsx
const navItems: NavItem[] = [
  // ...existing items
  {
    icon: <PieChartIcon />,
    name: "Reports",
    subItems: [
      { name: "Monthly", path: "/reports/monthly" },
      { name: "Annual",  path: "/reports/annual", pro: true },
    ],
  },
];
```

<Note>
  Icons are React components exported from `src/icons/index.tsx`. Add your own SVG to `src/icons/` and export it from the index file to keep imports consistent.
</Note>

## 4. Layout

The admin shell is assembled in `src/app/(admin)/layout.tsx`. It places `AppSidebar` on the left, `AppHeader` at the top, and renders page content in a constrained container:

```tsx
"use client";

import { useSidebar } from "@/context/SidebarContext";
import AppHeader from "@/layout/AppHeader";
import AppSidebar from "@/layout/AppSidebar";
import Backdrop from "@/layout/Backdrop";

export default function AdminLayout({ children }: { children: React.ReactNode }) {
  const { isExpanded, isHovered, isMobileOpen } = useSidebar();

  const mainContentMargin = isMobileOpen
    ? "ml-0"
    : isExpanded || isHovered
    ? "lg:ml-[290px]"
    : "lg:ml-[90px]";

  return (
    <div className="min-h-screen xl:flex">
      <AppSidebar />
      <Backdrop />
      <div className={`flex-1 transition-all duration-300 ease-in-out ${mainContentMargin}`}>
        <AppHeader />
        <div className="p-4 mx-auto max-w-(--breakpoint-2xl) md:p-6">
          {children}
        </div>
      </div>
    </div>
  );
}
```

Common layout adjustments:

- **Change the content max-width** — edit `max-w-(--breakpoint-2xl)` to another breakpoint token such as `max-w-(--breakpoint-xl)`.
- **Change the content padding** — edit the `p-4 md:p-6` classes on the innermost `div`.
- **Change the sidebar width** — update the `w-[290px]` and `w-[90px]` values in `AppSidebar.tsx` and the corresponding `lg:ml-[290px]` / `lg:ml-[90px]` margins in the layout.

## 5. Components

Every UI component in TailAdmin is a plain React component in `src/components/`. To modify a component, open its file and edit the JSX and Tailwind classes directly.

### Dark mode

All components use the `dark:` prefix to apply dark-mode styles. For example, the `ComponentCard` container:

```tsx
<div className="rounded-2xl border border-gray-200 bg-white dark:border-gray-800 dark:bg-white/[0.03]">
```

Add `dark:` variants alongside your light-mode classes when extending any component.

### Responsive variants

TailAdmin follows the mobile-first Tailwind convention. Responsive variants are applied with `sm:`, `md:`, `lg:`, and `xl:` prefixes:

```tsx
<div className="p-5 md:p-6">
```

### Custom utility classes

For styles that appear in many components, define a `@utility` in `globals.css` rather than duplicating classes:

```css
@utility stat-card {
  @apply rounded-2xl border border-gray-200 bg-white p-5
         dark:border-gray-800 dark:bg-white/[0.03] md:p-6;
}
```

Then use it like any other Tailwind class: `<div className="stat-card">`.
