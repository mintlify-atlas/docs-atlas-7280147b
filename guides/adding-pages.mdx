---
title: "Adding pages"
description: "How to create new pages inside the TailAdmin admin layout using the Next.js App Router."
---

Every page inside the admin shell lives in `src/app/(admin)/`. The `(admin)` folder is a Next.js route group — the parentheses prevent it from appearing in the URL. Any `page.tsx` file you create inside this group automatically inherits the sidebar and header from `src/app/(admin)/layout.tsx`.

## Create a new page

<Steps>
  <Step title="Create the route folder and page file">
    Create a new folder for your route inside `src/app/(admin)/`, then add a `page.tsx` file:

    ```
    src/app/(admin)/analytics/page.tsx
    ```

    This makes the page available at `http://localhost:3000/analytics`.
  </Step>

  <Step title="Write the page component">
    A basic page exports a default React Server Component and exports `metadata` for the browser tab title:

    ```tsx
    import PageBreadcrumb from "@/components/common/PageBreadCrumb";
    import { Metadata } from "next";

    export const metadata: Metadata = {
      title: "Analytics | TailAdmin - Next.js Dashboard Template",
      description: "Analytics overview page.",
    };

    export default function AnalyticsPage() {
      return (
        <div>
          <PageBreadcrumb pageTitle="Analytics" />
          <div className="rounded-2xl border border-gray-200 bg-white p-5 dark:border-gray-800 dark:bg-white/[0.03] md:p-6">
            <p className="text-sm text-gray-500 dark:text-gray-400">
              Your analytics content goes here.
            </p>
          </div>
        </div>
      );
    }
    ```
  </Step>

  <Step title="Add the page to the sidebar navigation">
    Open `src/layout/AppSidebar.tsx` and add an entry to `navItems` (or `othersItems`):

    ```tsx
    import { PieChartIcon } from "../icons/index";

    const navItems: NavItem[] = [
      // ...existing items
      {
        icon: <PieChartIcon />,
        name: "Analytics",
        path: "/analytics",
      },
    ];
    ```

    The sidebar automatically highlights the item when the current URL matches `path`.
  </Step>
</Steps>

## Using the `PageBreadcrumb` component

Import `PageBreadcrumb` from `@/components/common/PageBreadCrumb` and pass a `pageTitle` string. It renders the page heading and a breadcrumb trail back to Home:

```tsx
import PageBreadcrumb from "@/components/common/PageBreadCrumb";

<PageBreadcrumb pageTitle="Analytics" />
```

The component outputs:

```tsx
<div className="flex flex-wrap items-center justify-between gap-3 mb-6">
  <h2 className="text-xl font-semibold text-gray-800 dark:text-white/90">
    {pageTitle}
  </h2>
  <nav>
    <ol className="flex items-center gap-1.5">
      <li>
        <Link href="/">Home &rsaquo;</Link>
      </li>
      <li className="text-sm text-gray-800 dark:text-white/90">
        {pageTitle}
      </li>
    </ol>
  </nav>
</div>
```

## Adding a page with nested sub-routes

If your section has multiple pages, create a sub-folder and use `subItems` in the sidebar:

```
src/app/(admin)/reports/monthly/page.tsx
src/app/(admin)/reports/annual/page.tsx
```

```tsx
// src/layout/AppSidebar.tsx
import { ListIcon } from "../icons/index";

const navItems: NavItem[] = [
  // ...existing items
  {
    icon: <ListIcon />,
    name: "Reports",
    subItems: [
      { name: "Monthly", path: "/reports/monthly" },
      { name: "Annual",  path: "/reports/annual" },
    ],
  },
];
```

Clicking **Reports** in the sidebar expands the group and shows both sub-links. The active link is highlighted automatically based on the current pathname.

## Using additional route groups

The existing `(admin)` directory already contains nested route groups such as `(others-pages)` and `(ui-elements)`. These groups let you co-locate related pages without affecting the URL structure.

For example, the calendar page lives at:

```
src/app/(admin)/(others-pages)/calendar/page.tsx
```

but is accessible at `/calendar`. You can create your own nested groups the same way:

```
src/app/(admin)/(my-section)/settings/page.tsx  →  /settings
src/app/(admin)/(my-section)/billing/page.tsx   →  /billing
```

## Layout inheritance

You do not need to add `AppSidebar` or `AppHeader` to your pages. Any `page.tsx` inside `src/app/(admin)/` automatically uses the layout defined in `src/app/(admin)/layout.tsx`, which renders the sidebar, the backdrop overlay, the header, and the content wrapper.

```tsx
// src/app/(admin)/layout.tsx — already wires up the full shell
export default function AdminLayout({ children }: { children: React.ReactNode }) {
  // ...
  return (
    <div className="min-h-screen xl:flex">
      <AppSidebar />
      <Backdrop />
      <div className={`flex-1 transition-all duration-300 ${mainContentMargin}`}>
        <AppHeader />
        <div className="p-4 mx-auto max-w-(--breakpoint-2xl) md:p-6">
          {children}  {/* your page renders here */}
        </div>
      </div>
    </div>
  );
}
```
