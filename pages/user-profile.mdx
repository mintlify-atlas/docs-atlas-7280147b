The user profile page lives at `/profile` and is served by `src/app/(admin)/(others-pages)/profile/page.tsx`. It stacks three card components inside a white rounded panel, each of which opens its own `Modal` for inline editing.

## Page file

```tsx
// src/app/(admin)/(others-pages)/profile/page.tsx

import UserInfoCard from "@/components/user-profile/UserInfoCard";
import UserMetaCard from "@/components/user-profile/UserMetaCard";
import { Metadata } from "next";

  title: "Next.js Profile | TailAdmin - Next.js Dashboard Template",
  description:
    "This is Next.js Profile page for TailAdmin - Next.js Tailwind CSS Admin Dashboard Template",

          <UserMetaCard />
          <UserInfoCard />
          <UserAddressCard />
        </div>
      </div>
    </div>
  );
}
```

## Components

Each component follows the same pattern: display data in a read-only layout, expose an **Edit** button, and open a `Modal` containing a form pre-filled with the current values. The `useModal` hook manages open/close state.

## Connecting real user data

All three cards display hardcoded placeholder data. To populate them with real user information:

<Steps>
  <Step title="Fetch user data in the page component">
    The `Profile` page component is a React Server Component. Fetch the current user's data from your database or auth session and pass it as props to each card.

    ```tsx
    // src/app/(admin)/(others-pages)/profile/page.tsx
    import { getCurrentUser } from "@/lib/auth";

    export default async function Profile() {
      const user = await getCurrentUser();
      return (
        <div className="rounded-2xl border border-gray-200 bg-white p-5 dark:border-gray-800 dark:bg-white/[0.03] lg:p-6">
          <h3 className="mb-5 text-lg font-semibold text-gray-800 dark:text-white/90 lg:mb-7">
            Profile
          </h3>
          <div className="space-y-6">
            <UserMetaCard user={user} />
            <UserInfoCard user={user} />
            <UserAddressCard user={user} />
          </div>
        </div>
      );
    }
    ```
  </Step>
  <Step title="Add a user prop to each card">
    Update the component signatures to accept a `user` prop and replace the hardcoded strings with `user.firstName`, `user.email`, etc.
  </Step>
  <Step title="Implement the save handler">
    Replace `console.log("Saving changes...")` in `handleSave` with an API call or Server Action that persists the updated field values.
  </Step>
  <Step title="Handle avatar uploads">
    The avatar `<Image>` in `UserMetaCard` uses a static path (`/images/user/owner.jpg`). Replace this with a dynamic `src={user.avatarUrl}` and add a file upload flow to update it.
  </Step>
</Steps>

<Note>
All three card components are `"use client"` components because they use the `useModal` hook. If you need to load data inside them (e.g., in a Server Component), lift the data fetch to the `Profile` page and pass it down via props.
</Note>
