The sign-up page lives at `/signup` and is served by `src/app/(full-width-pages)/(auth)/signup/page.tsx`. It renders the `SignUpForm` component inside the same `AuthLayout` used by the sign-in page — a full-screen two-column layout with the form on the left and a branded dark panel on the right on large screens.

<Note>
Like the sign-in page, the sign-up page sits inside the `(full-width-pages)/(auth)` route group and therefore renders without the admin sidebar or header. See the [sign-in page](/pages/auth/sign-in) for details on the shared `AuthLayout`.
</Note>

## Page file

```tsx
// src/app/(full-width-pages)/(auth)/signup/page.tsx
```

## SignUpForm component

The form is a `"use client"` component at `src/components/auth/SignUpForm.tsx`. It manages `showPassword` and `isChecked` states locally, matching the pattern used in `SignInForm`.

## Customising the sign-up flow

<Steps>
  <Step title="Connect a real registration handler">
    The `<form>` has no `action` or `onSubmit` handler. Add your user-creation logic — for example, a Next.js Server Action that calls your database or a third-party auth provider SDK.

    ```tsx
    // Using a Next.js Server Action
    import { signUpAction } from "@/actions/auth";

    <form action={signUpAction}>
      {/* existing fields */}
    </form>
    ```
  </Step>
  <Step title="Add extra fields">
    To collect additional information (e.g., phone number, company), add a new `<div>` block inside the `space-y-5` container and use the `Label` + `Input` components following the existing pattern.
  </Step>
  <Step title="Enforce the checkbox">
    The `isChecked` state is managed locally but never validated. Add a check before submission to prevent form submission when `isChecked` is `false`.
  </Step>
  <Step title="Add validation">
    Use `react-hook-form` + `zod` for field-level validation. The `Input` component accepts a `ref` and is compatible with `register()`. Apply error messages below each field using a `<p className="text-error-500 text-sm">` element.
  </Step>
</Steps>

<Tip>
The Submit button in `SignUpForm` is a plain `<button>` rather than the shared `Button` component used in `SignInForm`. You can swap it for `<Button className="w-full" size="sm">Sign Up</Button>` to keep the two forms visually consistent.
</Tip>
