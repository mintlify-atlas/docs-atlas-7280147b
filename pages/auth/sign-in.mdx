The sign-in page lives at `/signin` and is served by `src/app/(full-width-pages)/(auth)/signin/page.tsx`. It renders the `SignInForm` component inside the `AuthLayout`, which shows a branded panel on the right-hand side on large screens and a floating theme toggler.

<Note>
Because this page is inside the `(full-width-pages)` route group, it renders without the admin sidebar or header. The `AuthLayout` (`src/app/(full-width-pages)/(auth)/layout.tsx`) provides its own full-screen two-column layout.
</Note>

## Page file

```tsx
// src/app/(full-width-pages)/(auth)/signin/page.tsx
import SignInForm from "@/components/auth/SignInForm";
import { Metadata } from "next";

export const metadata: Metadata = {
  title: "Next.js SignIn Page | TailAdmin - Next.js Dashboard Template",
  description: "This is Next.js Signin Page TailAdmin Dashboard Template",
};

export default function SignIn() {
  return <SignInForm />;
}
```

## SignInForm component

The form is a `"use client"` component located at `src/components/auth/SignInForm.tsx`. It manages two pieces of local state: whether the password field is visible (`showPassword`) and whether the "Keep me logged in" checkbox is checked (`isChecked`).

### What the form includes

<CardGroup cols={2}>
  <Card title="Social sign-in" icon="right-to-bracket">

  </Card>
  <Card title="Email field" icon="envelope">

  </Card>

## Customising the sign-in flow

<Steps>
  <Step title="Connect a real auth provider">
    The `<form>` element currently has no `action` or `onSubmit` handler. Add your authentication logic — for example, using [NextAuth.js](https://next-auth.js.org/) `signIn()` or a custom server action — to the `<form>` tag.

    ```tsx
    // Using a Next.js Server Action
    import { signInAction } from "@/actions/auth";

    <form action={signInAction}>
      {/* existing fields */}
    </form>
    ```
  </Step>
  <Step title="Wire up the social buttons">
    Replace the placeholder `<button>` elements with calls to your auth provider's OAuth flow, e.g. `signIn('google')` or `signIn('twitter')` when using NextAuth.
  </Step>
  <Step title="Add validation">
    Install a library such as `react-hook-form` + `zod` and wrap the inputs. The `Input` component accepts a standard `ref` prop and is compatible with `register()` from react-hook-form.
  </Step>
  <Step title="Handle the redirect">
    After a successful sign-in, redirect users with `redirect('/')` inside a server action, or `router.push('/')` from a client component using `useRouter`.
  </Step>
</Steps>

<Tip>
To add a third social provider button, duplicate one of the existing `<button>` elements in the grid and replace the SVG icon and label. The two-column `grid-cols-2` layout accommodates up to two buttons per row — add more by adjusting the grid class.
</Tip>
