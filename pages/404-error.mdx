---

TailAdmin includes two 404 implementations that share identical markup:

- **`src/app/not-found.tsx`** — The Next.js App Router [not-found convention](https://nextjs.org/docs/app/api-reference/file-conventions/not-found). Next.js renders this file automatically whenever `notFound()` is called inside a Server Component, or when no matching route is found.
- **`src/app/(full-width-pages)/(error-pages)/error-404/page.tsx`** — A navigable `/error-404` route inside the `(full-width-pages)` group, rendered without the admin sidebar or header.

<Note>
For automatic 404 handling in production, `not-found.tsx` is the file that matters. The `/error-404` route exists as a preview page within the template so you can inspect the design without triggering an actual missing route.
</Note>

## not-found.tsx (App Router)

Next.js automatically serves `src/app/not-found.tsx` when no route matches or when `notFound()` is called.

## Customising the 404 page

### Change the error message

Edit the paragraph text inside `not-found.tsx`:

```tsx
<p className="mt-10 mb-6 text-base text-gray-700 dark:text-gray-400 sm:text-lg">
  The page you requested does not exist. Please check the URL.
</p>
```

### Change the illustration

Replace `/images/error/404.svg` and `/images/error/404-dark.svg` with your own SVG or raster images. Update the `width` and `height` props on the `<Image>` components to match your image dimensions.

### Trigger not-found programmatically

Call `notFound()` from `next/navigation` inside any Server Component to render the `not-found.tsx` file:

```tsx
import { notFound } from "next/navigation";

export default async function ProductPage({ params }: { params: { id: string } }) {
  const product = await fetchProduct(params.id);

  if (!product) {
    notFound(); // Renders not-found.tsx
  }

  return <div>{product.name}</div>;
}
```

<Tip>
The `not-found.tsx` file at the root of the `app` directory acts as a global catch-all. You can also place `not-found.tsx` files inside nested route segments to provide segment-specific error pages with different messaging.
</Tip>
