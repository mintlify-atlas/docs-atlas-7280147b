## App Router route groups

Next.js route groups let you organise routes into folders without affecting the URL. TailAdmin uses two top-level groups.

### `(admin)`

All pages under `src/app/(admin)/` share the admin layout defined in `src/app/(admin)/layout.tsx`. This layout renders the `AppSidebar`, `AppHeader`, and a `Backdrop` overlay for mobile, then places page content in a responsive main area that shifts based on the sidebar state.

The `(admin)` group contains two nested sub-groups:

- **`(others-pages)`** — calendar, user profile, form elements, tables, charts, and the blank page template.
- **`(ui-elements)`** — individual showcase pages for each UI component (alerts, avatars, badges, buttons, images, modals, videos).

### `(full-width-pages)`

Pages under `src/app/(full-width-pages)/` use a separate layout that renders without the sidebar. This group contains:

- **`(auth)`** — sign in and sign up pages.
- **`(error-pages)`** — the custom 404 error page.

## Key configuration files

### `next.config.ts`

Configures the `@svgr/webpack` loader so that `.svg` files can be imported directly as React components in both Webpack and Turbopack modes.

```typescript
const nextConfig: NextConfig = {
  webpack(config) {
    config.module.rules.push({
      test: /\.svg$/,
      use: ["@svgr/webpack"],
    });
    return config;
  },
  turbopack: {
    rules: {
      '*.svg': {
        loaders: ['@svgr/webpack'],
        as: '*.js',
      },
    },
  },
};
```

### `postcss.config.js`

Uses the `@tailwindcss/postcss` plugin, which is the correct PostCSS integration for Tailwind CSS v4.

```javascript
module.exports = {
  plugins: {
    '@tailwindcss/postcss': {},
  },
};
```
