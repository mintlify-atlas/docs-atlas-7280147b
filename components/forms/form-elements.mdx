The **Form Elements** page (`/form-elements`) assembles every available form component into a two-column demo layout. It serves as a live reference for inputs, selects, checkboxes, radios, toggles, file uploads, and more.

## Components included

<CardGroup cols={2}>
  <Card title="Input" icon="text-cursor-input" href="/components/forms/input">
    Text, email, password, number, time, and payment card inputs with label, placeholder, error, success, and disabled states.
  </Card>
  <Card title="Select" icon="chevron-down" href="/components/forms/select">
    Single-option dropdown with a customisable placeholder and option list.
  </Card>
  <Card title="Date picker" icon="calendar" href="/components/forms/date-picker">
    Flatpickr-powered date picker with single, multiple, range, and time modes.
  </Card>
  <Card title="Multi-select" icon="list-checks" href="/components/forms/multi-select">
    Custom dropdown that lets users pick multiple options, displayed as removable tags.
  </Card>
</CardGroup>

## Available demo components

Each demo component lives in `src/components/form/form-elements/` and wraps one or more form primitives inside a `ComponentCard`.

| Component file | What it demonstrates |
|---|---|
| `DefaultInputs.tsx` | Text, placeholder, select, password (toggle visibility), date picker, time, and payment card inputs |
| `InputStates.tsx` | Error, success, and disabled states on the `Input` component |
| `SelectInputs.tsx` | `Select` and `MultiSelect` side by side |
| `TextAreaInput.tsx` | Default, disabled, and error `TextArea` |
| `InputGroup.tsx` | Email input with icon prefix; phone input with country-code dropdown at start and end positions |
| `FileInputExample.tsx` | `FileInput` with a label |
| `CheckboxComponents.tsx` | Default, checked, and disabled `Checkbox` |
| `RadioButtons.tsx` | Default, selected, and disabled `Radio` buttons |
| `ToggleSwitch.tsx` | Blue and gray `Switch` variants in default, checked, and disabled states |
| `DropZone.tsx` | Drag-and-drop upload zone powered by `react-dropzone` (accepts PNG, JPG, WebP, SVG) |

## Additional form primitives

Beyond the components shown in the demo page, the following primitives are available:

- **`Label`** (`src/components/form/Label.tsx`) — accessible `<label>` with consistent spacing and dark-mode colour.
- **`Checkbox`** (`src/components/form/input/Checkbox.tsx`) — controlled checkbox with checked SVG overlay.
- **`Radio`** / **`RadioSm`** (`src/components/form/input/Radio.tsx`, `RadioSm.tsx`) — controlled radio buttons in standard and compact sizes.
- **`TextArea`** (`src/components/form/input/TextArea.tsx`) — multi-line input with error and disabled states.
- **`FileInput`** (`src/components/form/input/FileInput.tsx`) — styled native file picker.
- **`Switch`** (`src/components/form/switch/Switch.tsx`) — toggle switch in blue or gray colour themes.
- **`PhoneInput`** (`src/components/form/group-input/PhoneInput.tsx`) — phone number field with country-code selector.
